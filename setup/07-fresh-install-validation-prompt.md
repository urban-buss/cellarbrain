# Fresh Install Validation — Agent Prompt

Copy-paste the prompt below into a fresh agent session to validate the PyPI
package end-to-end: install, ETL, and smoke tests.

---

## Prompt

```
You are validating a fresh install of the "cellarbrain" Python package from PyPI.
Follow these steps exactly and report the result of each step.

### Step 1 — Create a clean environment

```bash
mkdir cellarbrain-test && cd cellarbrain-test
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate
```

### Step 2 — Install from PyPI

Install core plus the dashboard and dev extras:

```bash
pip install "cellarbrain[dashboard,dev]"
```

Verify the install:

```bash
cellarbrain --version
```

Report the installed version.

### Step 3 — Prepare raw CSV files

Create a `raw/` directory and place the three Vinocell CSV exports inside:

```
raw/
├── export-wines.csv
├── export-bottles-stored.csv
└── export-bottles-gone.csv
```

These are exported from the Vinocell iOS app (Share → Export → CSV).
All three files are required. Tell me where to place them relative to the
project root, and confirm you can see them.

### Step 4 — Run the ETL pipeline

```bash
cellarbrain etl raw/export-wines.csv raw/export-bottles-stored.csv raw/export-bottles-gone.csv -o output
```

Report:
- Did it succeed?
- How many wines were processed?
- What files were created in `output/`?

### Step 5 — Run pytest (unit tests)

```bash
pytest
```

Report pass/fail counts. If any tests fail, show the failure summary.

### Step 6 — Run the full smoke-test pipeline

```bash
python -m tests.smoke_helpers --raw-dir raw --output-dir output
```

Note: The smoke pipeline expects timestamped sub-folders inside `raw/`
(e.g. `raw/260501/` containing the three CSVs). If you only have flat files,
create a dated folder first:

```bash
mkdir raw/260501
cp raw/export-*.csv raw/260501/
python -m tests.smoke_helpers --raw-dir raw --output-dir output --folders 260501
```

Report the smoke-test result and any failures.

### Step 7 — Verify MCP server starts

```bash
cellarbrain mcp
```

Confirm it starts without errors (Ctrl+C to stop).

### Step 8 — Verify dashboard starts

```bash
cellarbrain dashboard --port 8765
```

Confirm it starts and is reachable at http://localhost:8765 (Ctrl+C to stop).

### Summary

After all steps, give me a summary table:

| Step | Status | Notes |
|------|--------|-------|
| Install | ✓/✗ | version |
| ETL | ✓/✗ | wine count |
| pytest | ✓/✗ | pass/fail |
| Smoke tests | ✓/✗ | pass/fail |
| MCP server | ✓/✗ | |
| Dashboard | ✓/✗ | |
```

---

## Notes

- **Python requirement:** 3.11 or newer.
- **Raw files:** The three CSVs are the only input data. Everything else is
  generated by the ETL.
- **Optional extras:** If you also want the sommelier ML model, add
  `pip install "cellarbrain[sommelier]"` and run `cellarbrain train-model`
  after the ETL completes.
- **Config file:** Not required for basic validation. The ETL uses sensible
  defaults. For production use, create a `cellarbrain.toml` (see
  [settings reference](../docs/settings-reference.md)).
