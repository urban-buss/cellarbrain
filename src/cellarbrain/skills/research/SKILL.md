# Wine Research

Fact-only web research to populate dossier sections. Every claim must trace to a web source visited this session.

## Cardinal Rules

1. **Facts only.** Every claim from a page you visited. Never training data alone.
2. **Identify first.** Confirm winery + vintage + region before writing. Uncertain -> STOP.
3. **Skip over guess.** No data? Leave pending. Nothing > something wrong.
4. **One section at a time.** Separate `update_dossier` call per section.
5. **Never overwrite** populated sections unless explicitly asked.

## Confidence Gate

| Level | Criteria | Action |
|---|---|---|
| High | 2+ sources agree | Write |
| Medium | 1 reliable source | Write, note single-source |
| Low/None | Conflicts or no data | **SKIP** |

## Branch A: Per-Vintage Dossier

**Sections:** `producer_profile`, `vintage_report`, `wine_description`, `market_availability`, `ratings_reviews`, `tasting_notes`, `food_pairings`, `similar_wines`

1. `read_dossier(wine_id, sections=[])` -- identity + pending list
2. Web research: Wine-Searcher -> producer site -> Vivino (last resort)
3. Verify identity (winery + vintage + region + category match)
4. For each pending section with High/Medium confidence:
   `update_dossier(wine_id, section, content, agent_name="openclaw-research", sources=["url1", "url2"], confidence="high")`
5. `read_dossier(wine_id, sections=[])` -- confirm populated

**Batch:** `pending_research(limit=N)` or `pending_research(limit=N, include_stale=True)` for combined queue.

## Branch B: Companion Dossier (Tracked Wines)

**Sections:** `producer_deep_dive`, `vintage_tracker`, `buying_guide`, `price_tracker`

Note: Vintage Comparison matrix is auto-generated by ETL -- never write it manually.

1. `read_companion_dossier(tracked_wine_id, sections=[])` -- identity + pending
2. Web research (same source order as Branch A)
3. Verify identity (winery + wine name + category)
4. `update_companion_dossier(tracked_wine_id, section, content, agent_name="openclaw-research", sources=[...], confidence="high|medium")`

**Batch:** `pending_companion_research(limit=N)`

## Branch C: Refresh Stale Research

`stale_research(months=6, limit=10)` -- wines with outdated metadata.

For each result: re-research the section, then `update_dossier(...)` with fresh sources. The `date` in metadata auto-updates.

## Tools

| Tool | Purpose |
|------|--------|
| `read_dossier` | Load per-vintage dossier |
| `update_dossier` | Write agent sections (per-vintage) |
| `read_companion_dossier` | Load companion dossier (tracked wine) |
| `update_companion_dossier` | Write companion sections |
| `pending_research` | Priority queue of wines needing research |
| `pending_companion_research` | Priority queue of tracked wines needing research |
| `find_wine` | Resolve wine identity from text |
| `stale_research` | Sections older than N months |

## Output Format

Always pass `format="plain"` to every tool call. The user receives responses via iMessage where Markdown tables and formatting are not supported. Plain format uses numbered lists, bullet points, and simple text separators instead.
